The "Sphere of Spheres" Philosophy

Data isn't physical. It's spiritual. It's a graph.

⚛️

Atomic Meaning

The smallest unit of meaning isn't a string or an integer. It is an action (a verb) and its arguments (kārakas). ASHRU distills knowledge down to this absolute atomic truth.

🌌

Omnipresent Spheres

Stop forcing data into rigid folder hierarchies. In the ASHRU model, a concept is a Sphere. A Sphere can link to multiple other Spheres simultaneously, acting as a true Many-to-Many knowledge graph.

⚡️

Substrate Independent

Whether you are mapping a 500-line Java file, a movie script, or human chat history, ASHRU represents the actions in a single, universal topological format.

The ASHRU Specification (v1)

Every response starts with a version header, followed by line-typed records.

ASHRU/1
F|<file_path>          # optional: file marker (bulk ingestion)
S|<sphere_path>        # optional: sphere marker (multi-routing)
V|<verb columns 1-10>  # the verb itself, 10 pipe-separated columns

The 10 Columns of an ASHRU Verb (V|...)

# Column Description Required?
1verb_lemmaAtomic action (e.g., buy, deploy)Yes
2kartāAgent / SubjectOptional
3karmaObjectOptional
4karaṇaInstrument / ToolOptional
5sampradānaRecipientOptional
6apādānaSource / OriginOptional
7adhikaraṇaLocative / Where/WhenOptional
8tensep (past), n (now), f (future), c (conditional)Optional
9is_negated0 or 1 (false/true)Default 0
10attributesk=v;k=v (semicolon-separated)Optional
